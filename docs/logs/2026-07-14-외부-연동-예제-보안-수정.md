# 외부 연동 예제 최종 보안 수정

## 배경

외부 연동 예제의 최종 리뷰에서 iframe 내부 변경을 부모 `onInput`으로 감지한다고 오인한 UI, 업로드 후 commit 예외의 고아 object, 4바이트 ZIP signature에 의존한 검증, 공통/Next.js metadata 계약 차이가 발견됐다. 예제를 복사하는 서비스가 제공되지 않는 안전 보장을 믿지 않도록 코드와 문서를 함께 수정했다.

## 작업 내용

- dirty 상태와 이탈 경고를 제거하고 수동 저장 모델 및 저장 mutex를 적용했다.
- generation 기반 문서 전환/저장 결과 격리를 유지했다.
- HWPX 구조 validator를 repository의 필수 의존성으로 만들었다.
- commit 실패 후 cleanup, 원 오류 보존, durable GC queue 기록 계약을 추가했다.
- 파일명 decode와 canonical safe integer version/ETag 검증을 공통화했다.
- README에 validator 제한 예시와 실제 `onError` 동작을 기록했다.
