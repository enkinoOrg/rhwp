# 외부 RHWP 연동 가이드

## 배경

공용 RHWP Studio의 호스트 책임과 배포 정책은 이미 문서화됐지만, 외부 서비스 개발자가 실제 HWPX 조회, editor lifecycle, 저장 권한, Supabase version 관리를 한 흐름으로 확인할 단일 진입점이 없었다. 예제 코드와 운영 정책이 분산돼 있어 구현 계약을 놓치거나 RHWP 호스트에 문서 ID, 인증 token, signed URL을 전달할 위험이 있었다.

## 작업 내용

- 프레임워크 독립 최소 예제부터 Next.js App Router와 Supabase private Storage까지 연결하는 상세 가이드를 추가했다.
- `createRhwpDocumentSession`, `saveHwpxDocument`, `getDocumentFile`, `saveDocumentFile`, `DocumentRepository`, `SupabaseDocumentStorage`의 실제 계약을 파일 링크와 함께 기록했다.
- GET 200/401/403/404, PUT 200/400/401/403/409/413/500 계약, quoted ETag, append-only version, orphan 정리 정책을 명시했다.
- 50MB 선검사와 streaming 제한, fail-closed HWPX 구조 검증, 한글 파일명 header, editor lifecycle, 로컬 글꼴 정책을 운영 관점에서 정리했다.
- root README와 공용 호스트 아키텍처에서 가이드로 이동할 수 있게 연결했다.
- README의 기존 로드맵 링크를 실제 보관 경로로 바로잡았다.
- 후속 리뷰에서 확인된 commit 응답 유실, 누락 validator, GC 참조 재검사 문제를 TDD로 보완했다.
- 실제 코드 경로 `server/validate-hwpx-archive.ts`, `server/document-storage-gc.ts`와 Route/Component TypeScript fixture 검사를 가이드에 연결했다.
- 외부 연동 SDK 최소 버전을 보안 수정이 포함된 `0.7.19`로 명시했다.
- 후속 Critical 리뷰에 따라 `document_versions.operation_id` unique 계약과 동일 payload RPC 멱등 재호출을 추가했다.
- commit 응답 유실 시 같은 operation을 재시도하고, 재시도도 불명확하면 `operation_id`와 `not_before`를 durable queue에 기록하도록 바꿨다.
- [fast-xml-parser-adapter.ts](../../examples/nextjs-integration/server/fast-xml-parser-adapter.ts)와 secure policy adapter를 추가해 XML validation/parse 경계를 연결했다.
- 최종 Important 리뷰에서 필수 목록 밖 `Contents/section1.xml`이 검증되지 않는 문제를 확인해, archive의 모든 XML 계열 entry(`.xml`, `.hpf`, `.rdf`, `.rels`, `.opf`)를 제한된 bytes로 secure parse하도록 확장했다.

## 저장 및 보안 결정

- RHWP 호스트는 편집 UI만 제공하며 외부 서버가 매 요청 세션과 문서별 권한을 검증한다.
- 문서 ID, 인증 token, signed URL과 service role key는 RHWP 호스트 및 browser에 전달하지 않는다.
- Supabase private Storage에는 새 object만 업로드하고, RPC가 append-only version row와 current version을 한 트랜잭션으로 갱신한다.
- 경쟁 저장은 409을 유지하고 새 orphan object만 정리하며 기존 current object를 덮어쓰지 않는다.
- `commitNewVersion` throw는 commit-unknown으로 분류하고 같은 `operation_id`와 payload로 RPC를 재시도한다. 재시도도 불명확하면 지연 원 RPC를 고려해 즉시 cleanup하지 않는다.
- unknown object는 `not_before` grace period가 있는 durable GC queue에 남긴다. [document-storage-gc.ts](../../examples/nextjs-integration/server/document-storage-gc.ts)는 grace 이후 삭제 직전 operation 참조 여부를 다시 확인한다.
- [validate-hwpx-archive.ts](../../examples/nextjs-integration/server/validate-hwpx-archive.ts)는 proven ZIP library adapter의 `readBytes(maxBytes)`와 secure XML parser를 주입받아 제한을 fail-closed로 강제한다. 모든 XML 계열 entry(`.xml`, `.hpf`, `.rdf`, `.rels`, `.opf`)를 `fast-xml-parser`로 parse하고 `DOCTYPE`, `ENTITY`, malformed XML을 거부하며 `mimetype`은 XML parser에 전달하지 않는다.

## 알려진 한계

- `@rhwp/editor@0.7.19` 발행은 로컬과 GitHub Actions 모두 npm 인증이 없어 실패했다. 발행 전까지 외부 프로젝트는 문서에 명시한 고정 커밋의 SDK를 vendor해야 한다.
- 예제는 ZIP binary를 직접 parse하지 않는다. 각 서비스가 유지보수되는 proven ZIP library adapter를 `ZipInspector` interface에 연결해야 한다.
- 예제는 실행 가능한 Next.js 앱이나 실제 Supabase migration이 아니며, 각 서비스는 인증 helper, RLS, bucket 권한과 production 요청 흐름을 별도 검증해야 한다.
- RHWP Studio 테스트는 성공했지만 Node가 유효한 `--localstorage-file` 경로 없이 실행됐다는 기존 경고 1건을 출력한다.
